Running a Project With a Local App Server

The Google App Engine Launcher includes the App Engine local app server for running and testing your application on your computer. The local app server can run in "strict" mode, which disables the app server's automatic index configuration feature for verifying indexes. The app server can also be configured to clear its datastore each time it is started.

To run a project with the local app server:

  1. Select the project in the project list.

  2. Click the "Run" button.

    Or: Select the menu option Control > Run.

    Or: Press Control-R.

    The local app server for the project starts, and a green icon appears next to the project in the project list.

To stop the local app server:

  1. Select the project in the project list.

  2. Click the "Stop" button.

    Or: Select the menu option Control > Stop.

    Or: Press Control-Shift-R.

    The local app server stops, and the green icon next to the project in the project list disappears.

To run a project with the local app server using "strict" mode:

  1. Select the project in the project list.

  2. Select the menu option Control > Run Strict.

    Or: Press Control-Shift-R.

    The local app server for the project starts in strict mode, and a green icon appears next to the project in the project list. If the application performs a query that requires index configuration that isn't present in the index.yaml file, the application will return an error.

To clear the local datastore every time the local app server starts:

  1. Select the project in the project list.

  2. Select the menu option Edit > Application Settings...

    Or: Press Control-I.

    The application settings dialog opens.

  3. Enter --clear_datastore in the Extra Command Line Flags box.

  4. Click the "Update" button. The dialog closes.

  5. Run the app server as you normally would, stopping it first if necessary. Each time you start the app server, the local datastore will be cleared.

To preserve the local datastore between sessions of the local app server:

  1. Select the project in the project list.

  2. Select the menu option Edit > Application Settings...

    Or: Press Control-I.

    The application settings dialog opens.

  3. Remove any occurrences of "--clear_datastore" in the Extra Command Line Flags box.

  4. Click the "Update" button. The dialog closes. From now on, the datastore will not be cleared when the app server is run.
